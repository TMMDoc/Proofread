Notifications

Notification settings

Enable email notifications - This option is used to enable and configure the email notification.

Notification recipients - Enter recipient details, to enter more than one recipient use a space or comma to separate them.

Send messages in batch - This option is used to combine multiple messages and send them in batch. To avoid large messages, messages sent within a finite duration are combined.

Attach tooling simulation preview where applicable (Warning: Messages may become too large) - This option will add a tooling simulation preview to the email notification.

Mail settings

SMTP server - The SMTP email server address can be found in the account or settings section of the user’s mail client.

SMTP port - This setting is the basic standard that mail servers use to send emails to one another across the internet.

Transport protocol -This is used to send emails securely from the application to the mail server.

Server needs authentication - Enable this option for the server to be validated.

User-name - Enter the recipient’s email here.

Password - Enter the recipient’s password here.

Test…​ - Use the Test switch to check if an email has been delivered successfully to the user. An error message will be shown, as in the image below, if the test mail failed.

Mail notification does not work in interactive mode. So, editing a part, applying a change, and saving would not generate a notification being sent to the user, even if there is a tooling/design error in the part being saved.
Due to the nature of email communication, mail delivery can be much slower and therefore it is done in the background. Notifications may be lost if the engine is stopped/killed before all messages are delivered.
